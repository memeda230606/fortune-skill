# 奇门遁甲占问指南

本文件供 LLM 在解释 `scripts/qimen-chart.mjs` 输出时参考。脚本负责起局和结构化盘面，LLM 只负责解释、取舍和现实承接。

## 起局原则

- 奇门用于具体占问，不替代八字/紫微本命分析。
- 必须先确认问题、起局时间、起局地点；默认使用起局地真太阳时。
- `--time-basis standard` 只用于和外部排盘对照，正式解释优先采用 `true-solar`。
- 若 `timeCorrection.boundary.corrected.nearBoundary` 或 `solarTermBoundary.near` 为 true，应提示时间边界敏感，不把盘面写成唯一确定结论。

## 盘面读取顺序

1. 看 `ju`、`yuan`、`timeInfo.xunShou`、`timeInfo.voidness`，确认阴阳遁、局数、旬首和空亡。
2. 看 `zhiFu` 与 `zhiShi`，它们常代表盘面主轴、主导力量和事情入口。
3. 按占问主题取用神：
   - 事业/项目：开门、值使、日干、年命、相关宫位。
   - 财务/交易：生门、财务对应宫、天盘/地盘干生克。
   - 感情/合作：六合、乙庚、日干时干、世俗关系对应宫。
   - 出行/迁移：驿马、开门、天盘动象、内外盘。
   - 健康/压力：死门、病符类凶象、入墓、空亡；只作文化观察，不作医学判断。
4. 看九宫内的门、星、神、天盘干、地盘干、五行、旺衰、空亡。
5. 再看 `auspiciousPatterns`、`inauspiciousPatterns`、`gatePressure`、`liuYiJiXing`、`tombInfo`、`tenStemResponse`。

## 常用解释边界

- 吉格表示结构上有助力或路径，不等于事情必然成功。
- 凶格、门迫、击刑、入墓、空亡表示阻力、延迟、约束或条件不足，不等于必然失败。
- 值符/值使落空亡或受制时，应说明“主轴有虚、入口不稳、需要现实条件补足”。
- 天盘干克日干、门迫或入墓集中时，应强调沟通、流程、权限、预算、健康和时间成本。
- 不要把单个宫位、单个格局孤立断事；必须结合占问主题、用神、值符值使和现实背景。

## 输出要求

- 先说明时间校正和盘面主轴，再解释用神宫位。
- 结论使用“倾向、阻力、条件、可观察信号、建议边界”的格式。
- 不输出确定性事件、赌博/投资指令、医学诊断、法律结论。
- 结尾必须附免责声明：仅供传统文化研究和自我观察参考，不构成专业建议。
